# テスト手順

Taverna のテストは2層構成です。

| 層 | 対象 | 方法 |
|---|---|---|
| 自動（純粋ロジック） | CSV解析・esc/safeUrl・バッジ生成・フィルター判定など | `tests.html` をブラウザで開く |
| 手動（UI/地図/DB） | 画面操作・地図ピン・ページ遷移・実データ | 下のチェックリスト |
| 将来 | E2E（実ブラウザ操作の自動化） | Playwright（未実装・フェーズB） |

---

## 自動テスト（tests.html）

`tests.html` を**ブラウザで開くだけ**（Live Server 不要・オフライン可）。
純粋ロジックを自動チェックし、上部に「○ / ○ 件 合格」を表示します。全部緑ならOK。

カバー範囲：
- `utils.js`：`esc` / `safeUrl` / `fileExt` / `genId`
- `bulk.js`：CSV解析（`parseBulkText`／引用符・タグ分割・見出しスキップ・必須チェック）
- `seating.js`：`renderSeatingBadges`（カード/詳細・空時・エスケープ）
- `parking.js`：`renderParkingDetail`（台数・評価ラベル・メモ）
- `stores.js`：条件フィルター `CONDITION_FILTERS` の判定

> ロジックを変更したら、まず `tests.html` を開いて全緑を確認してからコミット推奨。

## 手動チェックリスト

ロジックでは確認できない「画面・地図・遷移・実データ」を目視確認する。
管理者でログインして実施（デモは閲覧専用）。

### 認証・デモ
- [ ] 管理者でログインできる
- [ ] デモログインで管理者の記録が**閲覧専用**で見える（追加/編集/削除ボタンが出ない）
- [ ] デモで `store-edit.html` に直接アクセスすると一覧へ戻される

### 一覧（index.html）
- [ ] カードが表示される／サムネイル・タグ・座席要点（ひとり◎等）・駐車場台数が出る
- [ ] 店名・エリア検索で絞り込める
- [ ] タグフィルター・条件フィルター（ひとり/子連れ/大人数/完全禁煙）が効く
- [ ] 「お店を追加」で登録ページへ遷移する

### 登録・編集ページ（store-edit.html）
- [ ] 新規登録：必須（店名・エリア）未入力で送信できない
- [ ] 新規登録：初回訪問ログ欄が出る／★評価を入れて登録すると詳細に訪問1件が入る
- [ ] 編集：`?id=` で各項目が埋まる／初回訪問ログ欄は出ない
- [ ] 駐車場を「あり/近隣にあり」で詳細欄（車サイズ・台数・メモ）が出る、「なし」で隠れる
- [ ] 保存後そのお店の詳細ページへ遷移する

### 訪問ログ・メニュー（store-detail.html）
- [ ] 訪問の追加・削除ができる
- [ ] メニューの追加・編集・削除、写真アップロード、サムネイル設定ができる
- [ ] 注文回数の ＋/− が効く

### 地図
- [ ] 住所検索（例：◯◯市◯◯町）で番地付近にピンが立つ（GSI）／見つからないとき町名に寄る
- [ ] 「店舗を置く/駐車場を置く」切替で赤/青ピンを設置できる、駐車場ピンを消せる
- [ ] 駐車場が「なし」だと駐車場モードが出ない／なしにすると青ピンが消える
- [ ] 詳細の「おみせはここ！」で店舗・駐車場の両ピンが表示される

### 一括登録（管理者・index.html）
- [ ] CSVを貼ってプレビュー → 必須未入力行が赤く除外される
- [ ] 「○件を登録」で一覧に反映される

## フェーズB：Playwright（将来）

実ブラウザでの操作自動化。導入時の想定：
- テスト用の Supabase プロジェクト（または専用テストユーザー）を用意し、本番データと分離
- 上の手動チェックリストのうち、ページ遷移・フォーム送信・フィルター・一括登録などをシナリオ化
- 地図タイルや外部ジオコーディングに依存する部分はモック or 寛容なアサーションにする
